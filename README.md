# 白菜杯 · 随机英雄

为 **白菜杯** 自定义赛事打造的英雄联盟随机选将工具。管理员创建对阵后，双方各 5 名队员在限定时间内从本方英雄池中选将，支持替换与队友协商交换。

**在线演示：** 部署于 [Render](https://render.com)（Web Service），推送 `main` 分支后自动更新。

## 项目简介

本工具用于解决娱乐赛 / 自定义对局中「随机英雄、公平选将、过程保密」的需求：

- 每场比赛由 **两支队伍** 对战，每队 **5 人上场**
- 开赛时为每队随机分配 **15 个不重复英雄** 作为本方英雄池（两队共 30 个，互不重复）
- 队员在 **120 秒** 内从本方池中选将，期间可随时替换
- 可与 **队友** 发起英雄交换，对方同意后生效（**不可与敌方交换**）
- 选将过程中 **队友互相可见**；**敌方选手、英雄池、选将信息全部隐藏**，倒计时结束后才公开

名单共 **6 支队伍、30 名队员**，另设 **管理员** 账号用于创建对阵与控制流程。

## 功能一览

| 功能 | 说明 |
|------|------|
| 免密登录 | 选择昵称即可进入（需在名单内） |
| 对阵管理 | 管理员选择两队、创建对阵、开始 / 重置选将 |
| 英雄池 | 每队 15 英雄，队内不重复，两队之间也不重复 |
| 倒计时选将 | 120 秒内选将、替换，时间到自动锁定 |
| 待选池 | 本方池中尚未被队友选用的英雄（紧凑缩略图展示） |
| 队友交换 | 向已选将的队友发起交换，需对方同意 |
| 敌方隐藏 | 选将倒计时内看不到任何敌方信息 |
| 管理员视图 | 可查看两队英雄池（含占用情况）及全员选将 |
| 实时同步 | 前端每 2 秒轮询，全员状态同步 |

## 使用流程

### 管理员

1. 登录时选择 **管理员**
2. 选择 **A 队** 与 **B 队** → 点击 **创建对阵**
3. 点击 **开始选英雄 (120s)**，系统为两队各分配 15 个英雄并启动倒计时
4. 实时查看两队 **英雄池** 与 **队员选将**
5. 选将结束或需要重开时，可点击 **重置场次**

### 上场队员

1. 选择自己的昵称登录
2. 等待管理员开始选将
3. 从 **本方英雄池（15）** 或 **待选池** 点击英雄完成选将
4. 倒计时内可随时 **替换** 当前选择
5. 若队友也已选将，可点击 **交换** 并等待对方同意
6. 倒计时结束前 **看不到敌方任何信息**；结束后公开双方阵容

### 未上场队员

可正常登录，若不在当前 10 人对阵名单中，页面会提示「本场未上场」。

## 本地运行

**环境要求：** Node.js 18+

```bash
git clone https://github.com/upsydaisy200353/baicai-cup-random-hero.git
cd baicai-cup-random-hero
npm install
npm start
```

浏览器打开 http://localhost:8765

英雄数据位于 `data/champions.json`，可通过 `scripts/fetch_champions.py` 从 Riot Data Dragon 更新。

## 部署（Render）

仓库包含 [`render.yaml`](render.yaml)，请使用 **Web Service**（Node.js），**不要** 使用 Static Site，否则 `/api/*` 接口无法工作。

1. 在 Render 连接本 GitHub 仓库
2. 选择 Blueprint 或手动创建 Web Service，`Build Command` 为 `npm install`，`Start Command` 为 `npm start`
3. 推送 `main` 分支后自动部署

可选环境变量：

| 变量 | 说明 |
|------|------|
| `SESSION_SECRET` | 登录 Token 签名密钥（生产环境建议修改） |
| `PORT` | 服务端口（Render 会自动注入） |

## 技术栈

- **前端：** 原生 HTML / CSS / JavaScript
- **后端：** Node.js + Express
- **状态：** 内存 + `data/runtime-state.json` 持久化；登录 Token 为无状态 HMAC 签名
- **英雄数据：** Riot Data Dragon API
